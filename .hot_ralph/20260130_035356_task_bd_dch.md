Implemented custom columns for the progress bar in `src/renderables/progress.zig`. The implementation adds:

### New Types

1. **`ColumnRenderContext`** - Contains all progress state (completed, total, percentage, elapsed, ETA, speed, etc.) passed to column renderers

2. **`ColumnWidth`** - Union for flexible width specification:
   - `.fixed` - Exact character width
   - `.ratio` - Proportional width allocation
   - `.auto` - Automatic sizing

3. **`BuiltinColumn`** - Enum of standard column types:
   - `description`, `bar`, `percentage`, `elapsed`, `eta`, `speed`, `spinner`, `completed`, `total`, `task_count`

4. **`ProgressColumn`** - The main column type with:
   - Builder pattern methods (`withWidth`, `withStyle`, `withMinWidth`, `withMaxWidth`, `hidden`)
   - Factory methods (`builtin()`, `custom()`, `text()`)
   - Render method that handles all column types

5. **`CustomColumnFn`** - Function signature for user-defined columns: `*const fn (ctx: ColumnRenderContext, allocator: Allocator) anyerror![]Segment`

6. **`Progress`** - New struct using column-based layout:
   - `withDefaultColumns()` - Description, bar, percentage
   - `withDownloadColumns()` - Full download progress (+ elapsed, speed, ETA)
   - `addColumn()` / `addCustomColumn()` for flexible composition
