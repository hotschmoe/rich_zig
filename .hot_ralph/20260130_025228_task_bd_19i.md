## Summary

I added pager support to `src/console.zig` with the following features:

### New Types

1. **`PagerOptions`** - Configuration struct with:
   - `use_external: bool` - Whether to try external pagers first (default: true)
   - `external_command: ?[]const u8` - Custom pager command to try
   - `prompt: []const u8` - Prompt shown at page breaks (default: ":")
   - `quit_keys: []const u8` - Keys that quit the pager (default: "qQ")
   - `scroll_lines: u16` - Lines to scroll per keypress (default: 1)

2. **`Pager`** struct with methods:
   - `init()` / `initWithOptions()` - Create a pager from a Console
   - `page(content: []const u8)` - Page text content
   - `pageSegments(segments: []const Segment)` - Page rendered segments

### Console Integration

- `console.pager()` - Get a pager with default options
- `console.pagerWithOptions(options)` - Get a pager with custom options
- `console.printPaged(content)` - Print content through the pager
- `console.printSegmentsPaged(segments)` - Print segments through the pager

### Behavior

1. If not a TTY, content is written directly to stdout without pagination
2. If `use_external` is true, tries external pagers (less/more on Unix, more on Windows)
3. Falls back to internal pager if no external pager is available
4. Internal pager shows one page at a time based on terminal height, with a prompt showing remaining lines
