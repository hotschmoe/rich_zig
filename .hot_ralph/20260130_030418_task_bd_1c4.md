## Summary

Created `src/logging.zig` with Rich-style log formatting:

### Core Types
- **Level** - Log levels (debug, info, warn, err) with conversions to/from `std.log.Level`
- **LevelStyles** - Customizable styles per level (debug=dim, info=blue, warn=yellow+bold, err=red+bold)
- **FormatOptions** - Configuration for timestamp format, level display, path display
- **LogRecord** - Structured log record with level, message, timestamp, optional path/line

### RichHandler
- Main handler for emitting styled log messages
- Builder pattern: `.setMinLevel()`, `.setColorSystem()`
- Convenience methods: `.debug()`, `.info()`, `.warn()`, `.err()`
- Configurable timestamp formats (time_only, datetime, iso8601)
- Path truncation support
- Color system awareness for terminal capability downgrading

### std.log Integration
- `stdLogFn` - Drop-in replacement for `std.options.logFn`
- `scopedLog(scope)` - Creates a scoped logger with Rich formatting

### Usage Examples
```zig
// Direct handler usage
var handler = rich.RichHandler.init(allocator);
defer handler.deinit();
_ = handler.setMinLevel(.info);
try handler.info("Application started");

// With std.log integration
pub const std_options = .{ .logFn = rich.logging.stdLogFn };

// Scoped logger
const log = rich.logging.scopedLog(.mymodule);
log.info("Processing item {d}", .{42});
```
